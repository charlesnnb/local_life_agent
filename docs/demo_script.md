# Demo Script (3-5 分钟)

## 开场 (30s)

> 我们做的不是本地生活推荐，而是本地生活任务执行 Agent。
>
> 传统推荐系统给你一个列表让你自己挑自己约。我们的 Agent 直接帮用户把事情做完——解析需求、规划路线、检查空位、预约下单、发消息，全部自动完成。

## 家庭场景 (60s)

> 小明是家住上海徐汇区的用户，有一个 5 岁的孩子，老婆最近在减肥。周六上午他打开 Agent，输入：
>
> "今天下午是空的，想和老婆孩子出去玩几个小时，别离家太远，帮我安排一下。"
>
> Agent 做了以下事情：
>
> 1. 识别这是家庭场景，提取约束：下午 2点出发，4-6小时，不超过8km，孩子5岁，要低脂餐
> 2. 搜索徐汇区的亲子活动场所和家庭餐厅
> 3. 逐一检查门票、空位、距离、排队时间
> 4. 筛选掉不适合 5 岁孩子的地方、距离超 8km 的地方
> 5. 选出最优方案：星河亲子探索乐园 + 轻氧家庭厨房
> 6. 自动完成：买门票、预约 3 人位、订低糖蛋糕、打车、发消息给家人

## 展示 Planning Trace (30s)

> 看 planning_trace 面板，能清楚看到每一步的决策过程：哪个 POI 被过滤了、为什么、fallback 是什么。

## 展示 Tool Calls (20s)

> 工具调用时间线展示了 7 次调用：门票查询 → 空位检查 → 排队检查 → 交通估算 → 创建门票订单 → 预约餐厅 → 订蛋糕 → 打车 → 发送消息。

## 朋友场景 (30s)

> 输入另一句："今天下午我们 4 个人，2 男 2 女，想出去玩几个小时，顺便吃饭，别太远。"
>
> Agent 自动识别为朋友场景，选择适合社交拍照的地点，生成可转发到群的消息。

## 亮点总结 (30s)

> 1. **不是推荐，是执行** — 完整闭环从需求到订单
> 2. **理解用户意图** — 家庭 vs 朋友场景自动识别
> 3. **9 阶段规划管线** — 每一步可解释可追溯
> 4. **完整 Fallback** — 无位、无票、太远、不合适都有应对
> 5. **轻量工程实现** — 无 LangChain，无真实 LLM，纯规则 + 评分 + Mock API，适合比赛快速演示
